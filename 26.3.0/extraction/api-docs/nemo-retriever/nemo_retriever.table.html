Run table-structure + OCR on table crops and produce structure-aware markdown.

For each row (page) in batch_df: 1. Read page_elements_v3 detections and page_image["image_b64"]. 2. Crop all table detections from the page image. 3. Run table-structure model on each crop to get cell/row/column bboxes. 4. Run OCR on each crop to get text with bboxes. 5. Join the two outputs using join_table_structure_and_ocr_output()

to produce properly-structured markdown tables.

  1. Fall back to OCR-only pseudo-markdown if table-structure returns no cells.

Parameters:

  • batch_df (pandas.DataFrame) – Ray Data batch with page_elements_v3 and page_image columns.

  • table_structure_model (NemotronTableStructureV1 | None) – Local table-structure model, or None for remote inference.

  • ocr_model (NemotronOCRV1 | None) – Local OCR model, or None for remote inference.

  • table_structure_invoke_url (str) – Remote NIM endpoint for table-structure inference.

  • ocr_invoke_url (str) – Remote NIM endpoint for OCR inference.

Returns:

Original columns plus table and table_structure_ocr_v1.

Return type:

pandas.DataFrame

Table extraction stage (pure Python + Ray Data adapters).

This stage enriches existing STRUCTURED/table primitives by populating metadata.table_metadata.table_content using OCR (and optionally YOLOX).
