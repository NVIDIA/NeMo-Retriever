nemo_retriever.local.stages.stage2_infographic_extraction module#

Local pipeline stage: infographic extraction wrapper.

This stage mostly delegates to nemo_retriever.infographic.stage, but adds a local-only convenience: –input may be either:

  • A single primitives DF file (e.g. <pdf>.pdf_extraction.json, .jsonl, .parquet)

  • A directory, in which case we iterate over all *pdf_extraction.json files in that directory and run the extractor on each.

This makes it easy to run stage2 across a whole folder of stage1 outputs.

nemo_retriever.local.stages.stage4_chart_extractor module#

Local pipeline stage: chart extraction wrapper.

This stage mostly delegates to nemo_retriever.chart.stage, but adds a local-only convenience: –input may be either:

  • A single primitives DF file (e.g. *.pdf_extraction.infographics.table.json, .jsonl, .parquet)

  • A directory, in which case we iterate over all stage3 table outputs in that directory (files ending with pdf_extraction.infographics.table.json) and run chart extraction on each.

This makes it easy to run stage4 across a whole folder of stage3 outputs.

If –web-ui is provided, starts a FastAPI web server for browser-based viewing.

Module contents#

Checkpointable local pipeline stages.

Each stage is a standalone Typer CLI module intended to be run independently, reading previous artifacts from disk and writing new artifacts alongside inputs.
