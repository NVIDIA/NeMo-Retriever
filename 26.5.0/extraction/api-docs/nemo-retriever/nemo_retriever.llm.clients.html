nemo_retriever.llm.clients.judge module#

LLM-as-judge scoring.

LLMJudge uses a strong LLM to score generated answers on a 1-5 scale against a ground-truth reference answer.

transport: LLMRemoteClientParams,
sampling: LLMInferenceParams | None = None,
)[source]#

Bases: object

LLM-as-judge that scores candidate answers on a 1-5 scale.

Configuration is split into two Pydantic objects:

  • transport: LLMRemoteClientParams owns the endpoint, api_key, retries, and timeout for the judge model.

  • sampling: LLMInferenceParams owns temperature / top_p / max_tokens. Defaults to temperature=0.0, max_tokens=256 for deterministic scoring.

nemo_retriever.llm.clients.litellm module#

Unified LLM answer generation client.

LiteLLMClient wraps the litellm library which provides a single interface for routing to NVIDIA NIM, OpenAI, HuggingFace Inference Endpoints, and local vLLM / Ollama servers via a model name prefix convention.

class nemo_retriever.llm.clients.litellm.LiteLLMClient(
transport: LLMRemoteClientParams,
sampling: LLMInferenceParams | None = None,
)[source]#

Bases: object

Unified LLM client backed by litellm.

A single model string change routes to any supported provider: - NVIDIA NIM: nvidia_nim/<org>/<model> - OpenAI: openai/<model> - Any OpenAI-compatible server (vLLM, Ollama): openai/<model> + api_base - HuggingFace: huggingface/<org>/<model>

Provider API keys are read from environment variables automatically (NVIDIA_API_KEY, OPENAI_API_KEY, HUGGINGFACE_API_KEY, etc.).

Configuration is split into two orthogonal Pydantic objects:

  • transport: LLMRemoteClientParams owns provider endpoint, authentication, retry, and timeout.

  • sampling: LLMInferenceParams owns temperature, top_p, and max_tokens.

Use from_kwargs() for a flat, backwards-compatible constructor.

Module contents#

Concrete LLM client implementations and a lightweight client registry.

The llm.clients package hosts every concrete LLMClient implementation in its own submodule (litellm.py, judge.py, …) so that adding a new client means adding a new file rather than extending a monolithic module. To keep the public import path stable, the registered client classes plus the internal prompt-helper (_build_rag_prompt) and response-parser (_parse_judge_response) are re-exported from this package’s namespace. Any caller that imports from nemo_retriever.llm.clients import LiteLLMClient will therefore continue to work unchanged after the module-to-package refactor.

nemo_retriever.llm.clients.register_client(name: str, cls: type) → None[source]#

Register a client class under a human-readable name.

The registry is optional: every concrete client remains addressable by its import path. Registration is offered as a convenience for configuration-driven instantiation (e.g. reading a type key from a YAML file and looking up the matching class).

Parameters:

  • name – Stable lookup name (e.g. "litellm").

  • cls – The client class. Must expose a from_kwargs classmethod to be useful to configuration-driven callers.
