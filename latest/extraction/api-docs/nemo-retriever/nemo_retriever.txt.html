nemo_retriever.txt.split module#

Tokenizer-based text splitting for .txt ingestion.

Produces chunk DataFrames compatible with embed_text_from_primitives_df and the LanceDB row builder (text, path, page_number, metadata).

) → DataFrame[source]#

Re-chunk a DataFrame’s text column by token count.

This is a post-extraction transform: it takes rows that already have a text column (produced by extract / extract_txt / etc.) and splits long texts into multiple rows using split_text_by_tokens(). All other columns (path, page_number, metadata, …) are preserved on every output row. Each chunk row’s metadata dict is updated with chunk_index and chunk_count.

Rows whose text is empty or missing are passed through unchanged.

Parameters:

  • df (pd.DataFrame) – Input DataFrame with at least a text column.

  • max_tokens – Forwarded to split_text_by_tokens() / _get_tokenizer().

  • overlap_tokens – Forwarded to split_text_by_tokens() / _get_tokenizer().

  • tokenizer_model_id – Forwarded to split_text_by_tokens() / _get_tokenizer().

  • tokenizer_cache_dir – Forwarded to split_text_by_tokens() / _get_tokenizer().

  • encoding – Forwarded to split_text_by_tokens() / _get_tokenizer().

Returns:

Expanded DataFrame (one row per chunk).

Return type:

pd.DataFrame

Split text into chunks by token count with optional overlap.

Chunk boundaries align with tokenizer token boundaries. Uses a sliding window: each chunk has at most max_tokens tokens; consecutive chunks overlap by overlap_tokens tokens.

Parameters:

  • text (str) – Input text to split.

  • tokenizer – HuggingFace tokenizer (e.g. AutoTokenizer) with encode/decode.

  • max_tokens (int) – Maximum tokens per chunk.

  • overlap_tokens (int) – Number of tokens to overlap between consecutive chunks (default 0).

Returns:

Chunk strings in order.

Return type:

list[str]

Read a .txt file and return a DataFrame of chunks (one row per chunk).

Columns: text, path, page_number (chunk index, 1-based), metadata. Shape is compatible with embed_text_from_primitives_df and LanceDB row build.

Parameters:

  • path (str) – Path to the .txt file.

  • max_tokens (int) – Max tokens per chunk (default 512).

  • overlap_tokens (int) – Overlap between consecutive chunks (default 0).

  • tokenizer_model_id (str, optional) – HuggingFace model id for tokenizer (default: same as embedder).

  • encoding (str) – File encoding (default utf-8).

  • tokenizer_cache_dir (str, optional) – HuggingFace cache directory for tokenizer.

Returns:

Columns: text, path, page_number, metadata.

Return type:

pd.DataFrame
