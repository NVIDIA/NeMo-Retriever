ocr_lang: Literal['multi', 'english'] | None = None,
) → str[source]#

Resolve public OCR selectors to the NemotronOCRV2(lang=...) selector.

Parameters:

  • ocr_version – "v1" for legacy English-only mode or "v2" (default) for multilingual mode.

  • ocr_lang – Optional language selector for v2. "multi" (default) or "english". Must be None when ocr_version is "v1".

Returns:

The lang string to pass to NemotronOCRV2(lang=...).

Raises:

ValueError – If ocr_version is not one of ['v1', 'v2'], or if ocr_lang is provided together with ocr_version='v1', or if ocr_lang is not one of ['multi', 'english'].

Invoke one or more image NIM HTTP endpoints with batched concurrent requests.

invoke_url may be a single URL or a comma-separated URL list. When multiple URLs are provided, batches are distributed round-robin.

Parameters:

  • merge_levels – Optional per-image merge level ("word", "sentence", or "paragraph"). When provided, must have the same length as image_b64_list. Passed as merge_levels in the JSON payload so the NIM can apply per-crop merging behaviour.

  • image (Returns one response item per input)

  • order. (in the same)

  • note:: (..) – Prefer NIMClient in long-lived actors to avoid creating a fresh ThreadPoolExecutor on every call.

nemo_retriever.ocr.shared.ocr_response_to_text(preds: Any) → str[source]#

Extract joined OCR text from a Nemotron OCR response, returning "" when no text is detected.

Wraps _parse_ocr_result() + _blocks_to_text() but suppresses _parse_ocr_result’s last-resort stringify fallback (which dumps the raw response repr when no shape matches) — that fallback produces noise rows for callers like the video frame OCR actor where many frames have no on-screen text and we need an empty-string sentinel to drop them.

nemo_retriever.ocr.shared.split_ocrable_rows(
batch_df: DataFrame,
ocrable_content_types: Sequence[str] = ('',),
) → Tuple[DataFrame, DataFrame][source]#

Partition rows into OCR-able rows and passthrough rows.

Rows whose _content_type is in ocrable_content_types are treated as OCR-able; all others are passed through unchanged. Batches with no _content_type column at all (audio-free pipelines like PDF / image) are fully OCR-able. The default accepts only the empty discriminator; pipelines that mix OCR-able and non-OCR-able rows (e.g. video, which interleaves video_frame and audio) pass their own sentinel set.
