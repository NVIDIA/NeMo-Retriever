Find the best matching online runner for the given resource requirements.

When multiple candidates match, jobs are distributed across them in round-robin order so that a batch of jobs doesn’t all land on the same machine.

preferred_runner_ids (list) takes precedence over the legacy scalar preferred_runner_id. When a list is provided the search is limited to those runners (round-robin among the online ones).
